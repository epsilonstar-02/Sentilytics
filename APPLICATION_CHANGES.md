# Application Architecture Changes

## 1. Overview
This document outlines the significant architectural evolution of the application. The system has transitioned from a file-based storage system to a robust relational database architecture, culminating in the implementation of high-performance **Vector Search (RAG)** to address scalability and performance bottlenecks.

## 2. Core Storage Migration (JSON → PostgreSQL)
**Previous State:**
- Data (transcripts, metadata) was stored as individual JSON files in the file system.
- Search operations required loading and parsing thousands of JSON files, which was slow and inefficient ("taking too much time").

**Current State:**
- **PostgreSQL Database**: Implemented a centralized relational database (`youtube_analytics`).
- **Schema Design**:
  - `products`: Stores product information (e.g., iPhone 17, Samsung S25).
  - `videos`: Stores video metadata (title, URL, publish date).
  - `transcripts`: Stores the full text of video transcripts.
- **Benefits**:
  - ACID compliance and data integrity.
  - Efficient SQL querying capabilities.
  - Foundation for advanced analytics and vector search.

## 3. Vector Search Implementation (RAG)
To further enhance search relevance and speed, the application now simulates **Google AlloyDB's** vector search capabilities using PostgreSQL.

### Database Enhancements
- **Extension**: Enabled `pgvector` extension.
- **New Table**: `transcript_chunks`
  - **Purpose**: Stores segmented parts of transcripts for granular retrieval.
  - **Columns**:
    - `video_id`: Foreign Key to `videos`.
    - `chunk_index`: Sequence order.
    - `content`: Text segment (~1000 chars).
    - `embedding`: 768-dimensional vector generated by Google's `text-embedding-004`.
  - **Indexing**: `IVFFlat` index added for fast approximate nearest neighbor search.

### Data Processing Pipeline
- **Script**: `data_collector/generate_embeddings.py`
  - **Logic**: Fetches transcripts from DB → Chunks text (Sliding Window) → Generates Embeddings (Google Gen AI) → Stores in DB.
  - **Performance**: Implements batching and rate limiting to handle large datasets (1,800+ videos).

## 4. Application Logic Updates
- **Search Tool**: `agents/chatbot/tools/transcript_search.py`
  - **Vector Search**: Implemented `search_transcripts_vector` using Cosine Similarity (`<=>`).
  - **Hybrid Approach**: Prioritizes semantic vector search, falling back to SQL-based keyword search if necessary.
  - **Result**: Search now understands context (e.g., "battery life" finds related concepts) rather than just exact string matches.
